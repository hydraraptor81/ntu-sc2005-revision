# OS & Computer System Operation Study Summary

## 1. Computer Organization & I/O
* **Device Controllers:** Each device controller is in charge of operations
    for a particular device type.
* **System Bus:** This is the channel through which devices and the CPU
    communicate.
* **Notifications:** Devices notify the CPU that I/O operations are done
    using an interrupt.

## 2. Interrupts
* **Interrupt-Driven OS:** Operating systems are considered interrupt-driven
    because if there are no interrupts, the OS will idle and not execute.
* **Major Steps of Interrupt Handling:**
    1. The OS saves the state of the current execution.
    2. It determines the appropriate interrupt service routine according to
       the specific interrupt type.
    3. The chosen interrupt service routine is then executed.



## 3. Interrupt-Driven I/O vs. DMA (Direct Memory Access)
* **Interrupt-Driven I/O:** Used for small size I/O and low-speed devices,
    resulting in more frequent interrupts.
* **DMA:** Used for block-based I/O and high-speed devices, resulting in
    fewer interrupts.



## 4. Key OS True/False Concepts
* **Privileged I/O:** All I/O instructions are privileged (True). They must
    go through the OS to ensure legality and correctness.
* **Memory Protection:** Given a base register of 0x1000 and a limit of
    0x1000, accessing memory location 0x1FFF will NOT generate a trap (False).
    The valid range is [base, base+limit-1], translating to [0x1000, 0x1FFF].
* **Real-time vs. Time-sharing:** Popular OSs like Windows and Linux are
    not real-time systems (False); they are usually time-sharing systems.
* **System Calls & Traps:** A system call always generates a trap (True).
    The hardware switches from user to kernel mode so the kernel can execute.

## 5. Multiprogramming vs. Multiprocessing
* **Multiprogramming:** Running multiple programs concurrently (even on a
    single-core CPU) to improve overall CPU utilization.
* **Multiprocessing:** Executing programs on a system with multiple
    processing cores to increase computing power via parallel architectures.
