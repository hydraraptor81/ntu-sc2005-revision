## Question 1: Address Translation (Paging vs. Segmentation)

Given a 10-bit logical address: `1000011011`. We need to translate this into a
physical address using two different memory management schemes.

### a) Paging Scheme
* **Page Size:** 128 bytes ($2^7$ bytes).
* **Logical Address Breakdown:** * **Offset:** The page size dictates the offset. Since $2^7 = 128$, the
        offset requires the last **7 bits**.
    * **Page Number:** The remaining bits form the page number. $10 - 7 =$
        **3 bits**.
* **Binary Split:** * Page Number: `100` (which is decimal 4).
    * Offset: `0011011`.
* **Translation:** Looking up Page 4 in the provided Page Table (Figure Q4a),
    the corresponding frame number is `01001`.
* **Physical Address:** Frame Number + Offset = **`010010011011`**.

### b) Segmentation Scheme
* **Maximum Segment Size:** 256 bytes ($2^8$ bytes).
* **Logical Address Breakdown:** * **Offset:** The max segment size dictates the offset. Since $2^8 = 256$,
        the offset requires the last **8 bits**.
    * **Segment Number:** The remaining bits form the segment number.
        $10 - 8 =$ **2 bits**.
* **Binary Split:** * Segment Number: `10` (which is decimal 2).
    * Offset: `00011011`.
* **Translation:** Looking up Segment 2 in the Segment Table (Figure Q4b),
    the base address is `010011010000`.
* **Physical Address:** Base + Offset = `010011010000` + `00011011`
    = **`010011101011`**.

## Question 2: Two-Level Paging System Architecture

Assume a logical address consists of **`x` bits** for the page number and
**`y` bits** for the offset. Each page table entry (PTE) is 4 bytes.

**a) Page Size:**
The offset determines the page size. With `y` bits for the offset, the page
size is **$2^y$ bytes**.

**b) Number of entries in the Page Table:**
The number of bits for the page number determines the total pages. With `x`
bits, there are **$2^x$ entries** in the logical address space.

**c) Two-Level Paging Outer Page Table Entries:**
* Total size of a single-level page table = $2^x \times 4$ bytes.
* Number of entries that can fit inside a single page = $2^y / 4 = 2^{y-2}$.
* To find the number of outer page table entries, we divide the total number
    of pages by the number of entries a single page can hold:
    $2^x / 2^{y-2} =$ **$2^{x - y + 2}$ entries** in the outer page table.
* **Logical Address Format:** * Outer Page Number: `x - y + 2` bits.
    * Inner Page Number: `y - 2` bits.
    * Offset: `y` bits.
