# Operating Systems: I/O and Disk Scheduling Tutorial

## Question 1: True/False Statements on I/O Subsystems

* **(a) Buffering can be used to improve I/O efficiency for files that**
  **are being written and re-read rapidly.**
  * **Answer:** False.
  * **Justification:** Buffering is primarily used to cope with device
    speed mismatches or transfer size mismatches between devices, not
    caching for rapid re-reads.

* **(b) Process will be in a waiting state after performing an I/O**
  **system call if non-blocking I/O is used.**
  * **Answer:** False.
  * **Justification:** When using non-blocking I/O, the process does not
    wait. It will be able to continue executing immediately after making
    the system call.

* **(c) Device drivers are part of the kernel I/O subsystem.**
  * **Answer:** False.
  * **Justification:** Device drivers are not considered part of the core
    kernel I/O subsystem; rather, they serve as the independent interface
    to the physical hardware controllers.

## Question 2: Multiprogramming I/O Performance

**Context:** A process reads one block of data at a time to a buffer
using synchronous I/O, then processes the data in a loop.

### a) Improving Performance
The performance of the given process can be significantly improved by
using **double-buffering** combined with **asynchronous I/O**.

**Improved Pseudocode:**
```c
buffer2 <- read a block of data from disk using asynchronous I/O;
while (not end of file) {
    while (I/O not over) do nothing; // Wait for current I/O
    buffer1 <- buffer2; // Swap buffers
    buffer2 <- read next block of data from disk using asynchronous I/O;
    process data in buffer1; // Process while next block is reading
}
