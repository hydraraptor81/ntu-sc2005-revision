# Operating Systems Tutorial: Deadlocks & Banker's Algorithm

## Question 1: True/False Statements on Deadlock

* **(a) In the dining-philosopher problem with five philosophers, if we**
    **allow at most four of them to be hungry simultaneously, deadlock may**
    **still occur.**
    * **Answer:** False.
    * **Justification:** Deadlock will not occur. With five chopsticks
        available and a maximum of four hungry philosophers, there will
        always be at least one chopstick left over. This guarantees that at
        least one philosopher can successfully acquire both chopsticks,
        eat, and then release them for others to use.

* **(b) It is impossible to have a deadlock involving only one single**
    **process.**
    * **Answer:** True.
    * **Justification:** Deadlock inherently requires circular wait and a
        hold-and-wait condition where a process is waiting for a resource
        held by *another* process. A single process cannot satisfy these
        conditions on its own.

* **(c) If a resource allocation graph contains a cycle, then a deadlock**
    **has occurred.**
    * **Answer:** False.
    * **Justification:** A cycle is only a sufficient condition for
        deadlock if each resource type has exactly *one instance*. If there
        are multiple instances per resource type, a cycle is a necessary
        but not sufficient condition (a deadlock *might* exist, but isn't
        guaranteed).

## Question 2: Resource-Allocation State Analysis

**Initial State (Available = 2):**

| Process | Allocation | Max | Need |
| :--- | :--- | :--- | :--- |
| **PROCESS 1** | 1 | 6 | 5 |
| **PROCESS 2** | 1 | 5 | 4 |
| **PROCESS 3** | 2 | 4 | 2 |
| **PROCESS 4** | 4 | 7 | 3 |

* **(a) If PROCESS 4 requests one unit of the resource next, does this**
    **lead to a safe or unsafe state?**
    * **Answer:** Unsafe.
    * **Explanation:** If we grant P4 one unit, its Allocation becomes 5
        and its Need becomes 2. The Available resources drop from 2 to 1.
        Looking at the new Needs (P1:5, P2:4, P3:2, P4:2), none of the
        processes can be satisfied with only 1 Available unit. The system
        will be trapped in an unsafe state leading to deadlock.

* **(b) If PROCESS 3 requests one unit of the resource next, does this**
    **lead to a safe or unsafe state?**
    * **Answer:** Safe.
    * **Explanation:** If we grant P3 one unit, its Allocation becomes 3
        and its Need becomes 1. The Available resources drop from 2 to 1.
        Because P3 only needs 1 more unit, it can execute and complete.
        Upon completion, P3 frees its allocated 3 units, making Available
        = 4. With 4 available, P2 can finish (needs 4), which then frees
        more resources allowing P1 and P4 to finish sequentially.
    * **Safe Sequence:** `<PROCESS 3, PROCESS 2, PROCESS 1, PROCESS 4>`

## Question 3: Banker's Algorithm with Multiple Resource Types

**Initial Snapshot:**
Available = `A:0, B:x, C:y, D:1`

| Process | Allocation (ABCD) | Need (ABCD) |
| :--- | :--- | :--- |
| **P0** | 2 2 0 0 | 1 4 0 0 |
| **P1** | 3 1 0 0 | 0 1 0 0 |
| **P2** | 1 1 0 0 | 1 0 0 0 |
| **P3** | 0 0 1 1 | 0 0 1 1 |
| **P4** | 0 0 1 1 | 0 0 3 2 |

**Scenario:** P3 requests `A:0, B:0, C:0, D:1`.
*Question:* What are the minimum values for `x` and `y` to grant this
request safely?

**Step 1: Apply P3's Request**
* P3's new Allocation = `0 0 1 2`
* P3's new Need = `0 0 1 0`
* New Available = `0 x y 0` (D drops from 1 to 0)

**Step 2: Determine Minimum `x` (Resources A & B for P0, P1, P2)**
* P1 needs `B=1`. To let P1 run first, `x` must be at least 1.
* If P1 runs, it releases `3 1 0 0`. Available becomes `3 (x+1) y 0`.
* P2 needs `A=1`, which is satisfied by Available `A=3`. After P2 runs,
    it releases `1 1 0 0`. Available becomes `4 (x+2) y 0`.
* P0 needs `B=4`. Current Available B is `x+2`.
* To satisfy P0, we need `x + 2 >= 4`, which means **minimum `x = 2`**.

**Step 3: Determine Minimum `y` (Resources C & D for P3, P4)**
* P3 needs `C=1`. To let P3 run, `y` must be at least 1.
* If P3 runs, it releases `0 0 1 2`. Available becomes `4 4 (y+1) 2`.
* P4 needs `C=3, D=2`. Current Available C is `y+1`.
* To satisfy P4, we need `y + 1 >= 3`, which means **minimum `y = 2`**.

**Conclusion & Safety Algorithm Verification:**
* **Min values:** `x = 2`, `y = 2`.
* **Initial Available after granting request:** `0 2 2 0`
* **Safe Sequence Generation:**

| Process | Available Before | Need | Allocation Freed | Available After |
| :--- | :--- | :--- | :--- | :--- |
| **P1** | 0 2 2 0 | 0 1 0 0 | 3 1 0 0 | 3 3 2 0 |
| **P2** | 3 3 2 0 | 1 0 0 0 | 1 1 0 0 | 4 4 2 0 |
| **P3** | 4 4 2 0 | 0 0 1 0 | 0 0 1 2 | 4 4 3 2 |
| **P0** | 4 4 3 2 | 1 4 0 0 | 2 2 0 0 | 6 6 3 2 |
| **P4** | 6 6 3 2 | 0 0 3 2 | 0 0 1 1 | 6 6 4 3 |

The resulting safe sequence is `<P1, P2, P3, P0, P4>`.
